apache2配置ftp server (mod_ftp)

wget:Linux下的无界面下载工具，支持http, ftp等协议。

1、下载apache2的ftp源码。
缺省apache2不包含mod_ftp，所以需要源代码，然后自己编译；取得源代码有两种方式：

  • 下载发行版本，然后解压出来

  • 从SVN server上同步最新版本

2、编译mod_ftp

  • 编译apache的模块，需要apache开发工具。

  • 切换到mod_ftp源码目录

在编译过程中由于conn_rec结构的remote_ip改成了client_ip，所以ftp_commands.c、ftp_data_connection.c、ftp_message.c、ftp_limitlogin.c这几个文件需要修改。

3、安装mod_ftp

4、配置mod_ftp
修改apache2.conf，注释到Include etc/apache2/extra/ftpd.conf，不在这里load ftp组件。

增加/etc/apache2/mods-available/ftpd.load配置文件。enable ftp组件。

ftpd.conf配置文件中有一个bug需要修复。

重新配置ftp根目录。

重启apache

把PuTTY当作Proxy 

今天注册了Google Cloud，有了一台Google Cloud的虚拟机。这台机器是在墙外面的，而且可以用PuTTY直接登陆上去。

本来想用这台机器直接作为一个代理，装了Apache2，研究了一下发现用Apache做代理还是要花一番功夫的。
手册上看，要启用Apache2的mod_proxy，需要先配置mod_ssl，mod_ssl又依赖openssl。也涉及到公钥/私钥的配置，
想一想感觉比较麻烦。PuTTY我已经配置好了公钥/私钥。网上找了一下PuTTY支持SSH远程登录的主机可以实现socks5代理
服务器的功能，而且配置也还方便。缺点就是不能保存。

下面是配置PuTTy。打开PuTTy,点击“Session”（打开默认就是此界面），输入要链接的地址、端口。

然后点击左侧“Connection”->“SSH”->“Tunnels”，输入一个端口号，比如：9999；选择Dynamic，然后点击Add。
再点击Open登陆主机。
浏览器中设置代理。FireFox，Options->Manual Proxy configuration->Socks Host:127.0.0.1 、Port : 9999、
Socks v5、Proxy DNS when using Socks v5。

这样设置完成。
